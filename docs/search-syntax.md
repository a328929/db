# 搜索语法指南

欢迎使用 TG Harvest 搜索功能！本指南将帮助你掌握强大的搜索语法。

---

## 🔍 基础搜索

### 精确匹配

默认情况下，搜索使用**精确匹配**：

```
福利      # 只匹配"福利"
onlyfans  # 只匹配"onlyfans"
iPhone    # 只匹配"iphone"（不区分大小写）
```

**中文搜索**：自动使用短语匹配，支持完整词组。

---

## 🎯 通配符搜索

如果需要模糊匹配，可以使用通配符：

### 星号 `*` - 匹配任意字符

```
test*     # 匹配以"test"开头的词：testing, tester, test123
*test     # 匹配以"test"结尾的词：mytest, pretest
*test*    # 匹配包含"test"的词：testing, contest, attest
```

### 问号 `?` - 匹配单个字符

```
te?t      # 匹配4个字符，中间任意：test, text, tent
h?t       # 匹配：hot, hit, hat
```

**提示**：如果不使用通配符，搜索会更快更精确！

---

## 🔤 布尔运算符

### AND 运算符：`+` 或 空格

查找**同时包含**多个词的消息：

```
福利+会员              # 必须同时包含"福利"和"会员"
onlyfans vip          # 空格等同于+，必须同时包含两个词
资源 下载 链接        # 必须包含全部三个词
```

**优先级**：最高（先计算）

### OR 运算符：`/`

查找**包含任一**词的消息：

```
福利/会员              # 包含"福利"或"会员"均可
telegram/discord      # 包含任一平台名即可
```

**优先级**：中等

### NOT 运算符：`-`

**排除**包含某词的消息：

```
福利-广告              # 包含"福利"但不含"广告"
onlyfans-免费          # 排除免费内容
资源-失效-过期        # 排除失效和过期的
```

**优先级**：高

**注意**：NOT可以连用实现双重否定：
```
--test                # 等同于 test
```

---

## 📋 复杂查询

### 括号分组

使用 `()` 改变运算顺序：

```
(福利/会员)+onlyfans              # "福利"或"会员"，且必须有"onlyfans"
资源+(视频/图片)-广告            # 必须有"资源"，视频或图片均可，排除广告
(免费/付费)+(telegram/discord)  # 两组词各选一个
```

### 短语搜索

用 `"双引号"` 精确匹配短语：

```
"Telegram 频道"       # 精确匹配整个短语
"onlyfans 福利"       # 词序必须一致
"免费 + 下载"         # 短语中的运算符会被当作普通字符
```

**短语中的特殊字符**会被当作普通文本处理。

---

## 🎬 视频时长搜索

使用 `【】` 或 `[]` 标记时长：

```
福利【01:30】         # 查找时长为1分30秒的视频
【10:00】             # 查找10分钟的视频
【1:05:30】           # 查找1小时5分30秒的视频
```

**格式**：
- `【MM:SS】` - 分:秒
- `【HH:MM:SS】` - 时:分:秒

**注意**：
- 时长必须**精确匹配**
- 一个查询只能包含一个时长条件
- 全角和半角括号都支持

---

## 📅 日期筛选

在搜索界面可以设置日期范围：

- **开始日期**：只显示此日期之后的消息
- **结束日期**：只显示此日期之前的消息

**格式**：
- `YYYYMMDD`：如 `20240101`
- `YYYY-MM-DD`：如 `2024-01-01`
- `YYYY/MM/DD`：如 `2024/01/01`
- `YYYY.MM.DD`：如 `2024.01.01`

可以按内容类型筛选：

| 类型 | 说明 |
|------|------|
| **搜索全部** | 所有类型的消息 |
| **文本** | 纯文字消息 |
| **图片** | 图片和照片 |
| **视频** | 视频、GIF、视频消息 |

## 📊 排序选项

### 按时间排序（默认）
- **降序**：最新的在前
- **升序**：最旧的在前

### 按相关性排序
只在有搜索词时可用，按匹配程度排序。

### 按文件大小排序
仅对图片、视频、音频可用。

### 按视频时长排序
仅对视频可用。

## 💡 搜索技巧

### 1. 从宽到窄
先用宽泛的关键词，看到结果后再加限定条件：

```
onlyfans          # 第一次搜索
onlyfans+福利     # 添加限定
onlyfans+福利-广告 # 排除无关内容
```

### 2. 善用群组筛选
如果结果太多，点击**群组标签**快速筛选到特定群组。

### 3. 使用通配符探索
不确定准确词语时：

```
*fans             # 找各种fans
tele*             # 找telegram相关
```

### 4. 组合搜索类型
先选择**视频类型**，再搜索关键词，缩小范围。

### 5. 利用排除词
快速过滤噪音：

```
资源-失效-过期-广告    # 一次排除多个
```

## ⚠️ 注意事项

### 搜索限制

1. **查询长度**：最多1000个字符
2. **分页限制**：最多浏览2000页（每页50条）
3. **频率限制**：
   - 普通搜索：20次/分钟
   - 后台统计：60次/分钟

### 性能建议

1. **精确匹配最快**：不用通配符时搜索更快
2. **避免过度通配**：`*test*` 比 `test` 慢
3. **先用范围筛选**：先选日期/类型/群组，再搜关键词

## 📚 运算符优先级

从高到低：

1. **括号** `()`
2. **NOT** `-`
3. **AND** `+` 或空格
4. **OR** `/`

### 示例

```
a+b/c-d           # 解析为：(a AND b) OR (c AND (NOT d))
(a+b)/(c-d)       # 解析为：(a AND b) OR (c AND (NOT d))
```

**建议**：复杂查询使用括号明确优先级。


## 💬 常见问题

### Q: 为什么搜索 `test*` 没有结果？
A: 确保使用英文星号 `*` 而不是中文 `＊`。

### Q: 如何搜索包含特殊字符的内容？
A: 使用双引号：`"100%完成"`

### Q: 搜索太慢怎么办？
A: 
1. 避免过多通配符
2. 先筛选日期/类型/群组
3. 使用精确匹配而非通配符

### Q: 为什么有些消息搜不到？
A: 
1. 确认关键词准确
2. 检查日期筛选范围
3. 尝试使用通配符：`*关键词*`

### Q: 如何查看更多结果？
A: 使用页面底部的翻页控件，或输入页码直接跳转。

## 🔧 环境变量配置（管理员）

如果你在反向代理（如Nginx）后部署，可以启用真实IP获取：

```bash
export TG_TRUST_PROXY=1
```

这将启用 X-Forwarded-For 头部解析，确保限流基于真实客户端IP。

**安全提示**：只在可信的反向代理环境中启用。
